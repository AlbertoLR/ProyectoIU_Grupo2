Descripción de las acciones de usuario con la interfaz del proyecto ET1/ET2 desde el punto de vista del usuario (desde el dominio del usuario).



Este es el conjunto de acciones que el usuario final desea realizar sobre la interfaz. No conforman en si, ni estructuran los requisitos funcionales ni la BD. Eso es lo que teneis que hacer vosotros en el proyecto. Lo que se pretende aquí es dar un espacio mínimo de acciones que el usuario quiere realizar con el sistema.

No esta estructurado de ninguna forma particular, sino según las iba identificando. Lo que debeis hacer es comprobar que vuestros requisitos funcionales y vuestro prototipo falso permiten hacer estas acciones y que vuestra base de datos permite almacenar toda la información que se maneja en estas acciones.
